# 🏦 ZK-Proof of Reserves (Institutional Architecture)

This repository contains a hybrid Proof-of-Concept (PoC) architecture for auditing banking liabilities using traditional cryptography and Zero-Knowledge Proofs (ZKP).

## 🎯 Project Objective
To demonstrate how a financial institution can mathematically prove on-chain that it holds sufficient reserves to cover client balances. This framework allows "ZK-VIP" clients to validate their solvency in a Web3 environment without revealing their actual balance or identity to the public.

## 🛠️ Technology Stack
* **Python & Web3.py:** Off-chain Merkle Tree construction integrating the Keccak-256 standard.
* **Circom & SnarkJS:** Development of ZK circuits (Groth16) and generation of cryptographic proofs using the military-grade **Poseidon Hash** algorithm.
* **Solidity:** Audit Smart Contract utilizing OpenZeppelin’s `MerkleProof` library.
* **Ethereum (Remix VM):** On-chain execution and verification environment.

## ⚙️ Operational Workflow
1. **The ZK Engine (Private Client):** A Circom circuit verifies that a user's vault balance meets a required minimum threshold (e.g., > $1,000 USD). It generates a unique Poseidon Hash, proving solvency without disclosing the actual account balance.
2. **The Hybrid Bank (TradFi + Web3):** A Python script builds a Merkle Tree by combining traditional clients (transparent balances) with the private client's ZK proof (Poseidon Hash), creating a cryptographically unbreakable Merkle Root.
3. **The Immutable Auditor (Smart Contract):** A Solidity contract stores the public Merkle Root. Any user can submit their cryptographic proof to the contract to verify in a decentralized manner that their funds are mathematically backed by the bank's reserves.

## 🚀 How to Run
1. Compile the circuit: `circom solvencia.circom --r1cs --wasm`
2. Generate the witness and proof using `snarkjs`.
3. Deploy `AuditorReservas.sol` in Remix and verify the proof against the Merkle Root generated by `merkle_banco.py`.
